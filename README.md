# Project-Transaction-Dashboard

# Studi kasus: Ritel
Product manager (PM) yang bersangkutan meminta bantuan kita untuk menganalisis performa penjualan. Untuk itu PM meminta sebuah dashboard dengan requirements sebagai berikut:
1. Menampilkan performa transaksi secara keseluruhan
2. Mengetahui trend penjualan
3. Mengetahui performa dari setiap metode pembayaran
4. Mengetahui product mana yang paling laku
5. Mengetahui spending untuk discount
6. Ringkasan transaksi untuk setiap produk dan konsumen

Pada dataset terdapat beberapa kolom, diantaranya yaitu:
1. id: kode order
2. customer_id: kode konsumen
3. order date: tanggal pemesanan
4. sku_id: kode produk
5. price: harga satuan produk
6. qty_ordered: kuantitas pemesanan
7. before_discount: harga sebelum diskon
8. discount_amount: total potongan
9. after_discount: harga setelah diskon
10. payment_id: kode dari tiap metode pembayaran
11. payment_method: nama metode pembayaran
12. sku_name: nama produk
13. base_price: harga dasar.
14. cogs: biaya perolehan atau biaya yang dibayarkan oleh perusahaan.
15. category: kategori dari tiap produk
16. registered_date: tanggal registrasi
